Ver.44

関税率調整方式

DDP手数料問題への新しいアプローチ

Ver.44の主な変更点

  • 販売価格計算を旧式（DDU基準）に戻しました → 競争力維持
  • 調整関税率を導入し、関税額でDDP手数料分を補填
  • 3%の安全係数で端数誤差をカバー
  • 実際の関税額を表示する列を追加（利益確認用）

背景：eBayのDDP手数料問題

eBayでは、DDP価格（販売価格＋関税）に対して手数料が課されるようになりました。つまり、関税額にも手数料がかかるため、従来の計算式では利益が目減りする問題が発生していました。

バージョン比較

Ver.42以前（旧式）

販売価格 = (仕入れ + 送料 + 利益) / (1 - 手数料 - 広告 - Payoneer) / 為替

想定関税 = 販売価格 × 実際の関税率 × (1 + 処理手数料率) + ...

問題点：DDP手数料が考慮されておらず、高額商品ほど利益が目減り

Ver.43（DDP手数料対応版）

販売価格 = (仕入れ + 送料 + 利益) / ((1 - Payoneer) - (1 + 関税率×(1+処理手数料率)) × (手数料 + 広告)) / 為替

問題点：販売価格が上昇し競争力が低下、既存出品の価格修正が必要

Ver.44（関税率調整方式）★今回

販売価格 = (仕入れ + 送料 + 利益) / (1 - 手数料 - 広告 - Payoneer) / 為替

調整関税率 = 実際の関税率 / (1 - 手数料率 - 広告費率) × 1.03

想定関税 = 販売価格 × 調整関税率 × (1 + 処理手数料率) + ...
項目 Ver.43 Ver.44
販売価格 上昇（競争力低下） 旧式と同じ（競争力維持）
計算式 複雑 シンプル（旧式）
利益確保の方法 販売価格で調整 関税額で調整
端数誤差対策 なし 3%安全係数

調整関税率の仕組み

eBayはDDP価格に対して手数料を課すため、関税額にも手数料がかかります。そこで、その手数料分を関税額に上乗せしてバイヤーから徴収します。

計算例

条件：実際の関税率 15%、手数料率 18%、広告費率 5%

調整関税率 = 15% / (1 - 0.18 - 0.05) × 1.03
             = 15% / 0.77 × 1.03
             = 19.48% × 1.03
             = 20.07%

この調整関税率（20.07%）で計算した関税額をバイヤーから徴収することで、DDP手数料分を補填します。

セル配置の変更

新規追加セル

セル 内容 説明
AA2 実際の関税率 ユーザーが入力（例: 0.15 = 15%）
AF2 調整関税率 自動計算（式: =$AA$2/(1-$F$1-$F$2)*1.03）
AH列 実際の関税額 参考表示用。AA2（実際の関税率）で計算

既存セルの変更

セル 変更内容
AD列 想定関税の計算にAF2（調整関税率）を使用

利益計算の確認方法

AD列（想定関税）とAH列（実際の関税額）の差額が、DDP手数料を補填するための上乗せ分です。

確認例（販売価格 $152.69 の場合）

項目 金額
想定関税（AD列） $31.28（調整関税率20.07%）
実際の関税額（AH列） $23.39（実際の関税率15%）
差額（上乗せ分） $7.89（約1,223円）

既存出品への対応

重要：レガシー再計算が必要

  • 販売価格の修正は不要 - Ver.42以前と同じ計算式のため
  • シッピングポリシーの再適用が必要 - 想定関税額が変わるため、シッピングポリシーの判定結果が変わる可能性があります
  • 一括シートから「レガシー再計算」を実行してください

適用手順

  1. 1スクリプトを更新 - Ver.44のコードをコピー
  2. 2初期設定を再実行 - または「計算式を再出力」を実行
  3. 3AA2に関税率を設定 - デフォルトは15%（0.15）
  4. 4レガシー再計算を実行 - 既存出品のシッピングポリシーを再適用

注意事項

  • AA2（実際の関税率）の設定を忘れずに - 商品カテゴリーに応じて変更してください
  • 3%の安全係数 - 端数処理や為替変動による誤差を吸収するため、調整関税率に3%の安全係数を掛けています
  • AH列（実際の関税額） - 利益確認用の参考値です。実際の利益計算に使用してください
