# 一括シートV3 シート・列・セル説明ドラフト

このファイルは開発中のメモ・意思決定を記録するためのドラフトです。
最終的に `sheet-reference.html` として記事化予定。

---

## 記事の構成案

1. 検索機能（シート名、列名、セル番地で検索）
2. シート一覧
   - 作業シート
     - 列説明（A〜AH列）

   - その他シート

## 作業シート 列説明（A〜AH列）

作成日: 2025-01-17

| 列 | 列名 | 説明 |
|----|------|------|
| A | 日付 | 入力日を記録 |
| B | 担当者 | 担当者の名前（外注さんを使っている方用） |
| C | label | 商品管理に使うSKU番号 |
| D | タグ | 商品のカテゴリや任意の仕分けに使う名前 |
| E | テンプレートID | 手動設定可、または翻訳実行時に自動入力。配送方法（X列）と状態（AE列）から判定されるため、それらが正しく入力されていることが前提 |
| F | 参考ebayID | EAGLEに登録の際に必須。詳しくはスクールの記事を参照 |
| G | 仕入れ先 | メルカリや楽天などの仕入れ先名 |
| H | 仕入れ先コード | 仕入れ先商品の商品コード |
| I | 仕入れ価格 | 仕入れ価格（数値のみ） |
| J | title | 商品の日本語タイトル（英語の場合もあり） |
| K | 商品説明 | 仕入れ商品のページに記載の商品説明 |
| L | セラーID | 仕入れ先のセラーのID |
| M | Title | 日本語タイトルと商品説明からAIが生成したeBay用英語タイトル |
| N | Condition/Description | 日本語タイトルと商品説明からAIが生成したeBay用商品説明・コンディション説明 |
| O | シッピングポリシー | 想定関税額（AD列）・配送方法（X列）・状態（AE列）から算出される固有のシッピングポリシーID。3要素が正しく入っていないと算出不可 |
| P | タイトル文字数 | 英語タイトルの文字数。80文字超でアラート（eBay上限80文字） |
| Q | 説明文の文字数 | 英語説明文の文字数 |
| R | 販売価格 | 各種設定値から算出されるDDU価格。eBay商品ページに登録される価格。送料・手数料込み、関税額のみ含まない |
| S | 関税込み価格 | 販売価格＋想定関税額。全て込みの参考価格 |
| T | 送料 | テーブル計算：サイズ・重量・配送方法から算出。送料固定：J1の値（空白時はProfit_Amountsシートから算出） |
| U | 利益 | 利益率設定：販売価格×利益率で算出。利益額設定：H1の値（空白時はProfit_Amountsシートから算出） |
| V | 手数料率 | F1の値を反映 |
| W | 利益率 | 利益率設定時：H2の値を反映（利益率計算のもと）。利益額設定時：空白 |
| X | 配送方法 | 仕入れ価格等から自動判定。判定基準はAJ2〜AJ5。テンプレート・シッピングポリシー判定に必須 |
| Y | 実重量 | 商品の重量。J2の値を反映 |
| Z | 長さ | 商品の長さ。L2の値を反映 |
| AA | 幅 | 商品の幅。M2の値を反映 |
| AB | 高さ | 商品の高さ。N2の値を反映 |
| AC | 容積重量 | 長さ・幅・高さから算出。クーリエとCpassエコノミーで算出方法が異なる。実重量との大きい方が送料算出基準 |
| AD | 想定関税 | 実際の関税＋関税手数料＋eBay関税手数料 |
| AE | 状態 | P2の値を反映。P2がAIの場合は翻訳時に自動判定。テンプレート・シッピングポリシー判定に必須 |
| AF | 基本送料 | 各配送方法の基本送料。サーチャージ・手数料を加えてT列の送料になる |
| AG | DDU調整後価格 | シッピングポリシーの関税上限超過時や、ゲーム等の送料上限超過時に関税額を上乗せした価格。超過時はこれが販売価格になる |
| AH | 実際の関税額 | 実際の関税率から算出される関税額＋関税手数料。Cpassから徴収される |

## 作業シート 固定セル説明

作成日: 2025-01-17

| セル | 名称 | 説明 |
|------|------|------|
| A2 | リアルタイム為替レート | 現在のリアルタイムの為替レート |
| C2 | 実効為替レート | この値が計算に使われる。1日1回の更新 |
| F1 | 手数料率 | ドロップダウンで選択可能。半角で入力必須 |
| F2 | 広告費率 | ドロップダウンで選択可能。半角で入力必須 |
| H1 | 利益額 | 利益額設定の時にこの値を利用。空白で利益額設定の場合はProfit_Amountsシートから算出 |
| H2 | 利益率 | ドロップダウンで選択可能。半角で入力必須 |
| J1 | 送料 | 送料固定計算の時にこの値を利用。空白で送料固定計算設定の場合はProfit_Amountsシートから算出 |
| J2 | 梱包重量 | 商品の梱包重量 |
| K1 | 価格計算ツールリンク | 価格計算ツールのリンク |
| L2 | 長さ | 梱包した商品の長さ |
| M2 | 幅 | 梱包した商品の幅 |
| N2 | 高さ | 梱包した商品の高さ |
| O1 | 送料上限カテゴリー | 商品の送料上限カテゴリー選択。ドロップダウンで設定。特定ジャンル（ゲーム、本など）は送料上限があるため必ず設定 |
| O2 | テンプレート名称設定 | 各テンプレートの名称を設定。ここで設定したものを元にテンプレートが判別される。ゲームやカードなど出品時は必ず切り替え。EAGLEに必要なテンプレートがないと正確に判別されない |
| P2 | 状態 | 商品の状態を選択。ドロップダウンで新品・中古・AIから選択。翻訳時に適用される。AI選択時は自動判定されるが精度は保証できず、出力されないことも多いため、手動で新品・中古を選択して翻訳することを推奨 |
| R1 | 配送方法（参考用） | ここで選択する配送方法はR2で計算する参考送料を算出するためのもので、実際の価格計算には影響しない |
| R2 | 参考送料 | シートの計算には影響しないが、参考の送料を算出できる |
| S2 | 容積重量 | 商品サイズから算出。クーリエとCpassエコノミーで算出式が異なる。郵便では使用しない |
| V1 | FedEx燃油サーチャージ | FedExの燃油サーチャージ |
| V2 | DHL燃油サーチャージ | DHLの燃油サーチャージ |
| W2 | Cpass割 | Cpass割 |
| Y1 | FedEx割増料金 | FedExの割増料金 |
| Y2 | DHL割増料金 | DHLの割増料金 |
| Z2 | ペイオニア手数料率 | ペイオニアの手数料率 |
| AA2 | 実際の関税率 | 実際の関税率 |
| AB2 | Cpassエコノミー通関手数料 | Cpassエコノミーの通関手数料 |
| AC2 | EU送料差額 | 最安値などを取りたい時に利用。詳しくはスクールに確認 |
| AE1 | 米国通関処理手数料 | 米国通関処理手数料 |
| AE2 | 米国VAT率 | 米国VAT率 |
| AF2 | 調整関税率 | 実際の関税率にeBayが徴収する関税に関する手数料を掛け合わせたもの |
| AG2 | 関税処理手数料 | 関税処理手数料 |
| AH2 | MPF | MPF |
| AJ2 | 低価格配送方法 | 初期設定で設定すると反映されるが、ここで変更すればシートに反映。変更後は式の再出力ボタンを押す |
| AJ3 | 高価格配送方法 | 初期設定で設定すると反映されるが、ここで変更すればシートに反映。変更後は式の再出力ボタンを押す |
| AJ4 | 送料切り替え基準（円） | 仕入れ価格をベースに配送方法を選択する基準価格。初期設定を反映するが、ここで変更した値がその場でシートに反映 |
| AJ5 | 送料計算方法 | テーブル計算か送料固定かを選択。初期設定で反映されるが、ここで変更後、式の再出力ボタンを押すことで式が入れ替わる |
| AL2 | 利益計算方法 | 利益の計算方法を選択。初期設定で反映されるが、ここで変更後、式の再出力ボタンを押すことで式が入れ替わる |
| AL3 | 手数料率 | 手数料率 |
| AL4 | 広告費率 | 広告費率 |
| AL5 | 利益率 | 利益率 |
| AN2 | 重量 | 重量 |
| AN3 | 長さ | 長さ |
| AN4 | 幅 | 幅 |
| AN5 | 高さ | 高さ |
| AP2 | DDU調整 | DDU調整の有効・無効の切り替え。常にONを推奨 |
| AP3 | 想定関税閾値 | 通常は390に設定。ゲームなど送料上限があるものは必ず切り替え（例：ゲームの場合は20） |
| AS2 | 使用プロンプト選択 | ドロップダウンで設定。デフォルトで各カテゴリごとのプロンプトがいくつか入っている |
| AT列 | 重複チェック | 重複チェックの式が入っている。既存商品と重複している場合「重複」と出力され、H列の仕入れ先コードが色づく |

## シート一覧

作成日: 2025-01-17

| シート名 | 説明 |
|----------|------|
| インポート用 | 拡張機能から直接データをインポートするシート。拡張機能で取得したデータや写真を確認できる |
| 内容確認専用シート | インポート用シートにインポートされたデータを並べ替えたもの。商品の評価や状態などがチェックしやすい |
| 作業シート | 実際に商品の計算や翻訳を行うシート |
| 出品用シート | EAGLEに登録する情報をまとめて、一括でコピーするためのシート |
| Import_Templates | EAGLEからテンプレートをインポートして判定するシート。作業シートで利用するために標準名を自動生成する |
| Import_Policies | EAGLEからシッピングポリシーをインポートして判定用に細分化するシート |
| Policy_Master | テンプレートとシッピングポリシーを判定用にまとめたもの |
| キーワード | 発見したキーワードを管理するシート。ユーザーが自由に設計 |
| キーワード管理 | 取得したキーワードの更新日や出品数、参考IDの管理をするシート。更新日のアラートを設定（K1の値）することで、更新から時間が経ったものを優先的に出品するという管理もできる |
| 参照データ | EAGLEに登録されているテンプレートや仕入れ先名称をダウンロード。ここのデータが作業シートのテンプレートと仕入れ先名のドロップダウンに反映される。また、各セルのアラートの色の設定も参考として記載 |
| 保存データ | 作業終了したデータの保存に活用。外注さんの作業記録などにも利用 |
| データ入力依頼シート | 外注さんに作業を依頼するのに活用 |
| EAGLE商品一覧 | EAGLEに登録されている商品の仕入れ先コードの管理に利用。重複管理用 |
| Profit_Amounts | 利益額と送料の設定に利用。利益額計算や送料固定の設定の場合、ここで管理 |
| GPT_Prompts | プロンプトの管理用シート。ここで直接編集もできるが、設定メニューから編集の方が使いやすい。新しいプロンプトを追加するときはA列にプロンプト名、B列に内容を入れる。作成後は初期設定を行うことで作業シートで使えるようになる（AS2の選択肢として反映） |
| Shipping_Methods | 配送方法の管理シート |
| Shipping_Rates | 送料テーブル表。各配送方法の送料テーブル |

## 意思決定ログ

### 2025-01-17: アシスタント機能の開発方針

- **決定**: 開発優先、記事は後でまとめて作成
- **理由**: 仕様変更の可能性があり、記事を先に作ると二重修正になる
- **運用**: 開発メモをこのファイルに記録、後で記事化
- **参考**: GPT/Gemini/Claudeの3者議論で合意

### 2025-01-17: フィードバック機能

- **決定**: カスタムフォーム → 既存Google Formへのリンクに変更
- **理由**: フィードバックの一元管理、既存フォームの活用
- **対応**: Google Formに「一括シートV3」選択肢を追加（ユーザー側作業）
